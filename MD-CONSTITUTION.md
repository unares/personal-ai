# .md Constitution Template

> Every .md file in Personal AI is a living constitution document.

## Required Sections

All .md files must include these sections:

### Mission Alignment
How this document serves **"Do One Thing. Earn Full Autonomy."**
- What is the One Thing this document enables?
- How does it contribute to the system earning autonomy?

### Scope
- What this document covers
- What it does NOT cover (explicit boundaries)

### Interfaces
- Who reads this document (agents, humans)
- Who writes/maintains this document
- What depends on this document

### Outcomes
Measurable results this document enables.

### Gamification Hooks
Opportunities for the future Gamification Agent. These are NOT implemented yet — they are pipework for rapid future implementation.

Format:
```
- [ ] {Metric}: {what it measures} → {what progression looks like}
```

### Document History
Living changelog. Update when the document evolves.

## Conventions

- **Terminology**: Always use "entity" (your portfolio or venture). Never use "company" or "project" in place of "entity".
- **Mission reference**: Every document should connect back to "Do One Thing. Earn Full Autonomy."
- **Living documents**: These evolve with the system. Review and update as code changes.
- **No implementation in gamification**: Only describe metrics and hooks. The Gamification Agent will implement the system later by reading all these hooks across all .md files.
